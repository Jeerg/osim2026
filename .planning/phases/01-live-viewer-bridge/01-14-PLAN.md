| Boundary | Description |
|----------|-------------|
| Engine-Kern → Listener | Listener lesen Engine-State read-only; ein versehentlicher Schreibzugriff (z. B. Mutieren von m_oProzCurrent) würde den Sim verfälschen. |

| Sim-Lauf → Reproduzierbarkeit | m_lPtkWartschl-Pflege darf die RNG-/Event-Reihenfolge nicht verändern. |

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-01-14-01 | Tampering | Listener lesen m_lRessBeleg | mitigate | Strikt read-only sampeln (keine Schreib-Ops); Zustands-Diff über id() statt Mutation am Engine-Objekt (Muster wie 01-01). |
| T-01-14-02 | Repudiation | m_lPtkWartschl-Pflege | mitigate | Eintragung/Austragung nur über reine Listen-Ops, keine Event-/RNG-Berührung; Determinismus-Pin im Integrationstest. |
| T-01-14-03 | Information Disclosure | auftrag_oid (Loader) | accept | OID ist eine modell-interne, nicht-sensible Lade-Id; kein PII. |
| T-01-14-SC | Tampering | npm/pip/cargo installs | accept | Keine neuen Pakete (stdlib + bestehendes jsonschema/pytest). Kein Audit-Gate. |
