<objective>
Bundle-Export und -Import implementieren. Ein Bundle ist die reproduzierbare Einheit eines Sim-Laufs: Modell-Definition + Sim-Config + Trace + KPIs + Manifest. Damit können User Läufe zwischen Tenants/Installationen austauschen (Beratungs-Kunde sendet Modell an Berater, Studierender reicht Klausur-Ergebnis ein, Forscher tauscht Benchmark-Modell mit Kollegen).

**Zweck:** Reproduzierbarkeitsvertrag (`seed, start_date, end_date, period_len`) bedeutet wenig, wenn nicht alle Inputs portierbar sind. Bundles machen einen Lauf vollständig reproduzierbar — der Empfänger kann denselben Trace bit-genau erzeugen.

**Output:** Export-Endpoint, Import-Endpoint, Import-Audit-Tabelle, UI-Buttons.

**Parallelisierbar:** Hängt NUR von Plan 01 ab. Wave 2 parallel zu Plan 02 und Plan 04.
</objective>

class BundleManifest:
  bundle_format_version: Literal["1.0"]
  created_at: datetime
  source_tenant_hash: str           # SHA-256 von tenant_id (nicht reversibel, nur für Konflikt-Detection)
  source_user_hash: str             # SHA-256 von Firebase-UID
  osim_ui_version: str
  engine_version: str
  kpi_schema_version: str
  contents:
    model_otx: bool                 # ist Modell als OTX enthalten?
    model_json: bool                # ist Modell als JSON enthalten?
    sim_config: bool
    trace: bool
    kpis: bool
  run_metadata:
    run_id_source: UUID             # Original-Run-ID (informativ)
    sim_params:
      seed: int
      start_date: str
      end_date: str
      period_len: int

# ZIP-Struktur:
# bundle.zip/
#   ├── manifest.json
#   ├── model.otx (optional, wenn Original-Format OTX war)
#   ├── model.json (optional, wenn JSON-Format aus Phase 2 verfügbar)
#   ├── sim_config.json
#   ├── trace.jsonl.gz (gzip-komprimiert, kann groß werden)
#   └── kpis.json

class BundleService:
  async def export_bundle(run_id: UUID, *, include_trace: bool = True) -> AsyncIterator[bytes]: ...

class BundleImporter:
  async def import_bundle(zip_bytes: bytes, target_tenant_id: str, importer_uid: str, *, replay: bool = False) -> ImportResult: ...

class ImportResult:
  bundle_import_id: UUID
  imported_model_id: UUID
  replay_run_id: UUID | None
  warnings: list[str]              # z.B. "Bundle-engine_version 0.3 < aktuelle 0.4 — Replay nicht bit-genau garantiert"
</interfaces>

    Service `app/services/bundle_service.py`:
    - `class BundleService`:
      - `__init__(self, run_service, model_service, kpi_service, storage_service, sim_config_service)`
      - `async def export_bundle(run_id: UUID, *, include_trace: bool = True) -> AsyncIterator[bytes]`:
        1. Lade Run + Sim-Config + Modell-Snapshot
        2. Berechne SHA-256 von tenant_id (`hashlib.sha256(tenant_id.encode()).hexdigest()`) und Firebase-UID
        3. Hole KPIs via `KPIService.compute_or_get`
        4. Falls `include_trace`: streame Trace-JSONL aus Storage, gzip-komprimiere via `gzip.GzipFile` in BytesIO oder direkt im ZIP-Eintrag
        5. Baue ZIP mit `zipstream-ng` (aus Plan 04 schon Dep) für Memory-effizientes Streaming
        6. Yield ZIP-Bytes
      - Wichtig: Manifest wird ZUERST in ZIP geschrieben (für Streaming-Konsumenten, die das Manifest sofort lesen können)

    Endpoint `app/api/v1/bundles.py`:
    - `POST /api/v1/runs/{run_id}/bundle` mit Query-Param `include_trace: bool = True`
    - `StreamingResponse` mit `Content-Disposition: attachment; filename="bundle-{run_id}.zip"`
    - Tenant-Auth wie immer
    - Rate-Limit (z.B. 5 Exports pro Minute pro User, via FastAPI-Middleware oder slowapi) — sonst Missbrauchspotenzial für große Trace-Files

    Code-Pflege:
    - Manifest enthält NICHTS Identifizierendes außer Hashes
    - `run_id_source` ist die UUID des Original-Runs — beim Import wird eine NEUE UUID vergeben, die `run_id_source` ist nur informativ ("dies stammt aus Lauf X")
    - Wenn das Modell als OTX existiert, hole es aus Storage und lege es ein. Wenn nur JSON existiert (Phase 2), lege JSON ein. Wenn beides, beide.

    Service `app/services/bundle_importer.py`:
    - `class BundleImporter`:
      - `__init__(self, model_service, run_service, storage_service, sim_orchestrator)`
      - `async def import_bundle(zip_bytes: bytes, target_tenant_id: str, importer_uid: str, *, replay: bool = False) -> ImportResult`:
        1. Validate ZIP-Struktur (manifest.json muss existieren)
        2. Parse Manifest → Pydantic-Validation, Version-Check
        3. Engine-Version-Vergleich → ggf. Warning
        4. Extrahiere model.otx ODER model.json (Bevorzugung: JSON, sonst OTX)
        5. Erstelle neues Modell in target_tenant_id via `ModelService.create_from_otx` oder `create_from_json`
        6. Sim-Config separat speichern oder mit Modell verlinken
        7. Wenn `replay=True`:
           - Starte neuen Sim-Lauf mit identischen Sim-Params (`seed`, `start_date`, `end_date`, `period_len`)
           - Trace landet in eigenem Storage-Pfad des Importers
        8. Persistiere BundleImport-Audit-Eintrag
        9. Return `ImportResult` mit Warnings

    - Berechtigung: jeder eingeloggte User darf in seinen eigenen Tenant importieren

    Klar dokumentieren:
    - Bit-Genauigkeit beim Replay ist nur garantiert, wenn engine_version + osim_ui_version + kpi_schema_version übereinstimmen
    - Bei Engine-Version-Diff: Replay wird mit Warning durchgeführt, aber kein Bit-Equality-Check erzwungen
    - Modell wird IMMER in Tenant des Importers kopiert (kein Modell-Sharing zwischen Tenants ohne Kopie)
