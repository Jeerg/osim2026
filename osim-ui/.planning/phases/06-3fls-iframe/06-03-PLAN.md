  <behavior>
    - `exchangeOsimToken(firebaseIdToken)` POSTet an `${VITE_OSIM_API_BASE_URL}/api/v1/auth/exchange-3fls-token`, returnt `{ custom_token, tenant_id, expires_in }` oder wirft `OsimExchangeError` mit `.status` und `.problem` (geparstes ProblemDetail-JSON falls vorhanden).
    - `useSimulationBridge({ iframeRef })` registriert window-message-Listener gefiltert auf `event.source === iframeRef.current?.contentWindow` UND `event.origin === VITE_OSIM_UI_BASE_URL`.
    - Hook-State: `status: 'idle'|'iframe-ready'|'auth-sent'|'error', lastHeight: number|null, lastError: string|null`.
    - Bei `ready`: `setStatus('iframe-ready')`.
    - Bei `resize.notify`: `setLastHeight(Math.max(600, payload.height))`.
    - Bei `auth.error`: `setStatus('error')`, `setLastError(payload.reason)`.
    - `sendAuth(token, tenantId)` ruft `iframeRef.current.contentWindow.postMessage({type:'auth.token', token, tenantId}, VITE_OSIM_UI_BASE_URL)` und setzt status='auth-sent'.
    - Origin-Mismatch und source-Mismatch werden verworfen ohne State-Update.
  </behavior>

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| 3fls-Frontend -> osim-ui-Backend | Cross-Origin-Fetch; osim-ui-CORS muss 3fls-Origin erlauben (in Welle 4 konfiguriert) |
| 3fls-Frontend -> osim-ui-Iframe (PostMessage) | targetOrigin MUSS auf osim-ui-URL eingeschränkt sein, NIE `"*"` |
| 3fls-Frontend window message-Listener | Eingehende Messages müssen origin- UND source-validiert sein |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-06.3-01 | Spoofing | window message-Listener im SimulationFrame | mitigate | useSimulationBridge filtert hart: `event.source === iframeRef.current?.contentWindow && event.origin === VITE_OSIM_UI_BASE_URL` |
| T-06.3-02 | Information Disclosure | postMessage mit custom_token an Iframe | mitigate | targetOrigin === VITE_OSIM_UI_BASE_URL, niemals `"*"`. Token niemals in iframe-src-Query und niemals in URL-History |
| T-06.3-03 | Tampering | Iframe-Code könnte manipuliert sein | mitigate | iframe `sandbox` ohne allow-top-navigation und ohne allow-popups; referrerPolicy no-referrer; CSP frame-src eng (Welle 4) |
| T-06.3-04 | Information Disclosure | Firebase-ID-Token im HTTP-Body | mitigate | TLS-Pflicht in Prod (Cloud-Run-Default); im Dev http://localhost akzeptiert |
| T-06.3-05 | Repudiation | Wer hat wann Simulation geöffnet? | accept | Audit-Log im osim-ui-Backend (Welle 2) deckt das ab; 3fls-Seite braucht kein zusätzliches Log |
| T-06.3-06 | Denial of Service | Iframe sendet Message-Flood | accept | osim-ui-Bridge debounced resize (Welle 1); Listener tolerant gegen hohe Frequenz, Cost ist State-Set — vernachlässigbar |
</threat_model>
