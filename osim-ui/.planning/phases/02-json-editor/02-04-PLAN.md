  <name>Task 1: Conversion-Service + Endpoint</name>
  <files>app/services/conversion_service.py, app/api/v1/conversion.py, tests/services/test_conversion_service.py, tests/api/test_conversion_endpoint.py</files>
  <behavior>
    - `convert_otx_to_json(...)`:
       1. Laedt das OTX-Source-Modell aus Storage (local file via `storage.get_local_path(otx_uri)`).
       2. Ruft `osim_engine.io.otx_loader.load_otx_file(path) -> LoadResult`.
       3. Ruft `osim_engine.io.json_dumper.dump_simulator_to_json(result.simulator) -> dict`.
       4. Baut `CoverageReport` aus `result.loaded`, `result.skipped`, `result.unsupported`, `result.warnings`, `result.coverage_ratio`.
       5. Persistiert ein NEUES Modell-Row mit `format='json'`, `payload_json=<dump>`, `name=f"{source.name} (JSON)"`. Setzt `schema_hash` via aktuellem Schema-Service.
       6. Verlinkt zurueck via Spalte `source_model_id UUID NULL FK models(id)` (Migration in diesem Task: `004_models_source_link.py`).
       7. Gibt `(new_model_id, coverage_report)` zurueck.
    - Idempotenz: pruefe ob bereits ein konvertiertes Modell mit `source_model_id == model_id` existiert. Wenn ja: ueberschreibe dessen payload statt neues Row anzulegen, gib dessen ID zurueck.
    - REST: `POST /api/v1/models/{id}/convert-to-json`. 201 mit `ConversionResponse`. 404 wenn source nicht im Tenant. 409 wenn `source.format != 'otx'`.
    - Tests Service: lade `Dummy.otx`, konvertiere, prueft `coverage_ratio > 0.8`, prueft idempotente Wiederholung -> gleiche `new_model_id`.
    - Tests Endpoint: konvertiert ein Test-Modell (Fixture aus Phase 1), erhaelt 201 mit Coverage-Report, GET auf `new_model_id` liefert JSON-Payload.
  </behavior>
  <action>
    Migration `004_models_source_link.py`: `op.add_column("models", sa.Column("source_model_id", UUID, nullable=True))` + `op.create_foreign_key("fk_models_source", "models", "models", ["source_model_id"], ["id"])`.

    Performance-Hinweis: `Bosch2_wechseln.otx` (18 MB) braucht laut Engine-API ~10 s OTX-Parse. Konvertierung sollte daher async laufen (FastAPI-Endpoint `async def` mit `await loop.run_in_executor(None, ...)` um den blockierenden Engine-Call). Timeout 120 s pro Konvertierung.
