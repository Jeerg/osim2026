## 4. Datenstrukturen

### OGPositionGrid (inc/GraphObj.h Z. 215)
Doppelt verlinkte Datenstruktur, **per-Objekt 4 Pointer + 2 Header**:
```c
class OGPositionGrid : public OGPosition {
    OGPositionGrid *pColNext;   // Spaltennachfolger
    OGPositionGrid *pColPrev;   // Spaltenvorgänger
    OGPositionGrid *pRowNext;   // Zeilennachfolger
    OGPositionGrid *pRowPrev;   // Zeilenvorgänger
    GOGridCol      *pColHead;   // Listenkopf der Spalte
    GOGridRow      *pRowHead;   // Listenkopf der Zeile
    CPoint          pGridPos;   // (x, y)-Position im Raster
};
```

**Topologie:** Jede Zelle des Rasters wird durch genau ein `OGPositionGrid` repräsentiert. Sie ist gleichzeitig
- in der Spalten-Doppelliste (sortiert nach `pGridPos.y`)
- in der Zeilen-Doppelliste (sortiert nach `pGridPos.x`)

Beide Listen sind **zirkulär** — der Spalten-Header (ein speziell ausgezeichnetes `OGPositionGrid` mit `pObj == LNULL`) ist gleichzeitig Anfang und Ende der Liste. Iteration: `while (pos->pObj != LNULL)`.

## 5. Memory-Management: Pool-Allokator für OGPositionGrid

Aus `OGGrid.cpp` Z. 487–538.

### `OGraphGrid::NewPos(colprev, colnext)` — neuen Knoten anfordern
```pseudocode
if (s_setPosFree == NULL):
    allokiere neuen Block (CPlex, 1000 × sizeof(OGPositionGrid))
    verkette alle 1000 in s_setPosFree (in umgekehrter Reihenfolge)

pPos = s_setPosFree                          // Kopf der Freelist
s_setPosFree = s_setPosFree->pColNext        // Freelist um 1 vorrücken
pPos->pColPrev = colprev
pPos->pColNext = colnext
pPos->pObj = NULL
m_objCount++
return pPos
```

### `OGraphGrid::FreePos(pos)` — Knoten zurückgeben
```pseudocode
pos->pColNext = s_setPosFree    // in Freelist einhängen
s_setPosFree = pos
m_objCount--
```

**Port-Hinweis:** In TypeScript brauchen wir keinen Pool — `new OGPositionGrid()` + GC reicht. Aber das verhalten muss identisch sein (gleiche pColPrev/pColNext-Initialisierung).

## 6. Listen-Köpfe Auto-Expand: `GetColHeadPos` / `GetRowHeadPos`

Aus `OGGrid.cpp` Z. 545 (`GetRowHeadPos`, analog GetColHeadPos Z. 646).

### Algorithmus `GetRowHeadPos(gnum)`
```pseudocode
m_isGridVEnlarged = FALSE
listcount = m_GRowList.count

// Falls Zeile gnum noch nicht existiert: Header bis dorthin anlegen
if (listcount <= gnum):
    m_isGridVEnlarged = TRUE
    for (i = listcount; i < gnum+1; i++):
        row = new GOGridRow()
        ogpos = NewPos(NULL, NULL)
        ogpos->pRowPrev = ogpos                 // zirkulär auf sich selbst
        ogpos->pRowNext = ogpos
        ogpos->pColPrev = NULL
        ogpos->pColNext = NULL
        ogpos->pObj = LNULL                      // SENTINEL
        ogpos->pRowHead = row
        ogpos->pColHead = NULL
        ogpos->pGridPos = (0, gnum)
        row->m_GRowPos = i
        row->m_GRowHeight = m_csStdGridExtent.cy
        row->m_OGPositionGrid = ogpos
        m_GRowList.AddTail(row)

    // alle neuen Zeilen StartPos/EndPos berechnen
    prevrow = NULL
    for each row in m_GRowList:
        if (i >= listcount):                     // nur die neu eingefügten
            if (prevrow == NULL):
                row->m_StartPos = m_cpStdGridOrigin.y
                row->m_EndPos = row->m_StartPos + row->m_GRowHeight
            else:
                row->m_StartPos = prevrow->m_EndPos + m_iStdLinkPlace
                row->m_EndPos = row->m_StartPos + row->m_GRowHeight
        prevrow = row

    m_isInsert = TRUE

// Jetzt suche den row-Header mit m_GRowPos == gnum
row = find row in m_GRowList with row->m_GRowPos == gnum
if (row == NULL): return NULL
if (row->m_OGPositionGrid->pRowNext->pObj == LNULL): return NULL   // leere Zeile
return POSITION(row->m_OGPositionGrid->pRowNext)                    // erstes echtes Objekt
```

**Kernidee:** Auto-Expand. Wenn ein Knoten an Position (x, y) eingefügt wird und Zeile y noch nicht existiert, werden alle Zeilen 0..y mit Header-Sentinels angelegt. Gleiches für Spalten.

## 7. ZENTRAL: `GOIns(obj, x, y, repaint)` — Knoten einfügen

Aus `OGGrid.cpp` Z. 1300–1491. **Dies ist das Herzstück der Rasteranordnung.**

```pseudocode
// 1. Koordinaten validieren (0 ≤ x, y < 1_000_000 oder == -1 erlaubt)
if (out of range): return FALSE

first = FALSE

// 2. SPALTEN-Einfügen
pos = GetColHeadPos(x)            // garantiert dass Spalte x existiert
ColHead = NULL
pNewGrid = NULL
insert = FALSE

if (pos == NULL):                  // Fall 1: Spalte ist leer
    ColHead = find col in m_GColList with m_GColPos == x
    if (ColHead == NULL): return FALSE  // darf nicht passieren
    pNewGrid = NewPos(ColHead.OGPositionGrid, ColHead.OGPositionGrid)
    ColHead.OGPositionGrid.pColPrev = pNewGrid
    ColHead.OGPositionGrid.pColNext = pNewGrid
    pNewGrid.pColHead = ColHead
    pNewGrid.pObj = obj
    pNewGrid.pGridPos.x = x
    first = TRUE
else:                              // Fall 2: Spalte nicht leer
    pGridRemembered = pos
    while (pos != NULL):
        gobj = GetColNext(pos)
        pGrid = (OGPositionGrid*) gobj.GetOGPositon()
        if (pGrid.pGridPos.y < y): pGridRemembered = pGrid
        if (pGrid.pGridPos.y == y): return FALSE   // Platz belegt!
        if (pGrid.pGridPos.y > y):
            // davor einhängen
            pNewGrid = NewPos(pGrid.pColPrev, pGrid)
            pGrid.pColPrev.pColNext = pNewGrid
            pGrid.pColPrev = pNewGrid
            pNewGrid.pColHead = pGrid.pColHead
            pNewGrid.pObj = obj
            pNewGrid.pGridPos.x = x
            insert = TRUE
    if (insert == FALSE):    // am Ende anhängen
        pNewGrid = NewPos(pGridRemembered, pGridRemembered.pColNext)
        pGridRemembered.pColNext.pColPrev = pNewGrid
        pGridRemembered.pColNext = pNewGrid
        pNewGrid.pColHead = pGridRemembered.pColHead
        pNewGrid.pObj = obj
        pNewGrid.pGridPos.x = x

// 3. ERSTE Objekt-Initialisierung
obj.m_OGPosition = pNewGrid
obj.m_OGView = m_OGView
obj.m_OGCollection = this

// 4. ZEILEN-Einfügen (spiegelsymmetrisch zu Schritt 2)
pos = GetRowHeadPos(y)
insert = FALSE
if (pos == NULL):                  // Fall 1: Zeile leer
    RowHead = find row in m_GRowList with m_GRowPos == y
    if (RowHead == NULL): return FALSE
    pNewGrid.pRowNext = RowHead.OGPositionGrid
    pNewGrid.pRowPrev = RowHead.OGPositionGrid
    RowHead.OGPositionGrid.pRowPrev = pNewGrid
    RowHead.OGPositionGrid.pRowNext = pNewGrid
    pNewGrid.pRowHead = RowHead
    pNewGrid.pGridPos.y = y
    first = TRUE
else:
    pGridRemembered = pos
    while (pos != NULL):
        gobj = GetRowNext(pos)
        pGrid = (OGPositionGrid*) gobj.GetOGPositon()
        if (pGrid.pGridPos.x < x): pGridRemembered = pGrid
        if (pGrid.pGridPos.x == x):
            // Doppel-Eintrag! Spalten-Eintrag rollbacken
            FreePos(pNewGrid)
            obj.m_OGPosition = NULL
            obj.m_OGView = NULL
            obj.m_OGCollection = NULL
            return FALSE
        if (pGrid.pGridPos.x > x):
            pNewGrid.pRowNext = pGrid
            pNewGrid.pRowPrev = pGrid.pRowPrev
            pGrid.pRowPrev.pRowNext = pNewGrid
            pGrid.pRowPrev = pNewGrid
            pNewGrid.pRowHead = pGrid.pRowHead
            pNewGrid.pGridPos.y = y
            insert = TRUE
    if (insert == FALSE):
        pNewGrid.pRowNext = pGridRemembered.pRowNext
        pNewGrid.pRowPrev = pGridRemembered
        pGridRemembered.pRowNext.pRowPrev = pNewGrid
        pGridRemembered.pRowNext = pNewGrid
        pNewGrid.pRowHead = pGridRemembered.pRowHead
        pNewGrid.pGridPos.y = y

// 5. Pixel-Position des GObject setzen
objorg.x = pNewGrid.pColHead.m_StartPos + m_GOrg.x
objorg.y = pNewGrid.pRowHead.m_StartPos + m_GOrg.y
obj.GetSize(size)
obj.SetPosition(objorg)

// 6. Invalidate falls nötig
if (repaint):
    if (size.cx > pNewGrid.pColHead.m_GColWidth
        || size.cy > pNewGrid.pRowHead.m_GRowHeight
        || first
        || m_isInsert):
        InvalidateView()
        m_isInsert = FALSE
    else:
        m_OGView.GetOGCtrl().InvalidateRect(NULL)

// 7. Notifikation
obj.OnGObj2CollAdded()
if (m_sendMessage and gctrl != NULL):
    gctrl.GetParent().SendMessage(WM_GOBJ_OBJ_INSERTED, NULL, (LPARAM)obj)

// 8. Subobjekt-Spezialbehandlung
if (obj is GObjSub and state == D_CLOSED):
    obj.SetChildsVisible(FALSE)

return TRUE
```

**Mini-Funktion `GOPIns(obj, p, repaint)`** ist nur ein Wrapper: `return GOIns(obj, p.x, p.y, repaint)`.

**Original-Bug (im Code geerbt):** Die `while (pos != NULL)` Schleifen in Schritt 2 und 4 haben in C++ KEIN umschließendes `{ }` — sie iterieren nur über die ERSTE Zeile danach. Das hat im Original entweder durch Glück funktioniert oder ist tatsächlich ein Bug. **Beim Port als richtige Schleife mit Block portieren** und gegen die Test-Modelle (Dummy.otx Durchlaufpläne) verifizieren.

## 9. `IsGridPlaceTaken` und `IsGridBlockTaken`

Aus Z. 1551, 1588.

```pseudocode
IsGridPlaceTaken(p):
    if (p.x >= colCount): return FALSE
    if (p.y >= rowCount): return FALSE
    // Spalte durchgehen
    pos = GetColHeadPos(p.x)
    while (pos != NULL):
        gobj = GetColNext(pos)
        pGrid = gobj.GetOGPositon()
        if (pGrid.pGridPos.y == p.y): return TRUE
    // Zeile durchgehen
    pos = GetRowHeadPos(p.y)
    while (pos != NULL):
        gobj = GetRowNext(pos)
        pGrid = gobj.GetOGPositon()
        if (pGrid.pGridPos.x == p.x): return TRUE
    return FALSE

IsGridBlockTaken(from, to):
    for (y = from.y; y <= to.y; y++):
        for (x = from.x; x <= to.x; x++):
            if (IsGridPlaceTaken(x, y)): return TRUE
    return FALSE
```

**Note:** Die DOPPELTE Prüfung (Spalte UND Zeile) ist defensiv — wenn die Datenstruktur konsistent ist, müsste die erste Prüfung ausreichen. Beim Port: erste Prüfung reicht, aber als Invariante dokumentieren.

## 11. `RemoveCol(x)` / `RemoveRow(y)` — Spalte/Zeile entfernen

Aus Z. 1799 (Col) und Z. 1865 (Row).

```pseudocode
RemoveCol(x):
    // VORBEDINGUNG: Spalte muss leer sein
    if (NOT IsColEmpty(x)): return

    erasecol = find col with m_GColPos == x
    if (erasecol == NULL): throw OException()

    // vorhergehendes Element holen
    pos = m_GColList.Find(erasecol); m_GColList.GetPrev(pos)
    oldgcol = pos ? m_GColList.GetAt(pos) : NULL

    // alle folgenden Elemente runternummerieren + Pixel-Pos updaten
    pos = m_GColList.Find(erasecol); m_GColList.GetNext(pos)
    while (pos != NULL):
        gcol = m_GColList.GetNext(pos)
        gcol.m_GColPos = x
        gcol.m_StartPos = oldgcol ? oldgcol.m_EndPos + m_iStdLinkPlace
                                  : m_iStdLinkPlace
        gcol.m_EndPos = gcol.m_StartPos + gcol.m_GColWidth
        // alle Objekte in spalte: pGridPos.x = x
        for each obj in gcol: obj.m_OGPosition.pGridPos.x = x
        oldgcol = gcol
        x++

    // erasecol aus der Liste nehmen und delete
    m_GColList.RemoveAt(m_GColList.Find(erasecol))
    delete erasecol
    if (repaint): InvalidateView()
    SendMessage(WM_GOBJ_GRID_COL_REMOVED, removecol, this)

    row = GetGRowAtPoint(p)
    if (col == NULL || row == NULL): return NULL
    return GetGOAtGrid(col.m_GColPos, row.m_GRowPos)

GetGridAtPoint(p, inGrid):
    // Pixel-Koords → Grid-Koords; mit Bounds-Check wenn inGrid==TRUE
```

`GetGColAtPoint(p)` und `GetGRowAtPoint(p)` durchsuchen die jeweilige Liste nach
`(StartPos <= p.x <= EndPos + iStdLinkPlace)` bzw. analog für y.

## 17. Test-Strategie

Für jeden portierten Algorithmus genau das C++-Verhalten verifizieren:

1. **GOIns-Sequenz-Test:** Knoten an (0,0), (1,1), (2,0), (1,0) einfügen → erwartete pGridPos-Werte UND pColNext/pColPrev/pRowNext/pRowPrev-Verlinkung exakt verifizieren.
2. **Auto-Expand-Test:** GOIns(obj, 5, 3) auf leerem Grid → m_GColList hat 6 Spalten, m_GRowList 4 Zeilen.
3. **Double-Insert-Test:** GOIns(obj1, 1, 1), GOIns(obj2, 1, 1) → zweiter muss FALSE liefern UND Spalten-Eintrag von obj2 muss rollbacked sein.
4. **InsertColBefore-Test:** 3 Knoten in 3 Spalten, InsertColBefore(1) → alle GColPos und pGridPos.x korrekt um 1 erhöht, StartPos/EndPos korrekt neu berechnet.
5. **RemoveCol-Test:** muss vorher IsColEmpty prüfen, sonst no-op.
6. **GetNextFreeGridPlace-Test:** Sweep-Reihenfolge (zeilenweise) ist im Original definiert.
7. **Pixel-Position-Test:** Nach GOIns(obj, 2, 1) muss obj.m_VirtRect.left = ColHead[2].m_StartPos + m_GOrg.x sein.

## 18. Offene Design-Fragen für die Implementierung

1. **GLNULL als Sentinel** — in TS als `Symbol` oder als spezielles Singleton-Object? Pragma: Singleton-Object mit `__sentinel: true`-Flag, damit `obj === LNULL` als O(1)-Vergleich funktioniert.
2. **GObject-Renderer-Trennung** — das Original mischt Domain (pGridPos, m_OGCollection) und View (Draw, m_VirtRect) in einer Klasse. Wir trennen in: `GObject` (reine Domain-Klasse, testbar ohne DOM) + `GObjectView` (React-Komponente die ein GObject rendert).
3. **MFC-CDC vs. React-Flow** — alle `Draw()`-Methoden im Original schreiben auf einen Device-Context. Im Port werden sie zu reinen `getRenderProps()`-Methoden, die den State liefern. React-Flow rendert.
4. **MoveMe / Drag-and-Drop** — das Original hat tracker-basierte Interaktion. React-Flow hat eigene Drag-Handler. Wir mappen 1:1 die Resultate (neue Position), nicht die Mechanik.
5. **WM_GOBJ_*-Notifications** — als Event-Emitter (`emit("obj-inserted", obj)`) implementieren statt SendMessage.
6. **Friend-Klassen-Zugriffe** — in TS werden protected-Member über package-interne Helper-Funktionen offengelegt; oder einfach alles `public` mit `// @internal`-Markierung.

## 20. Wichtige Invarianten zum Schutz

1. `pGridPos` eines `OGPositionGrid` MUSS jederzeit konsistent zu pColHead.m_GColPos und pRowHead.m_GRowPos sein.
2. Die Spalten- und Zeilen-Listen sind ZIRKULÄR (Header zeigt auf sich selbst wenn leer).
3. `pObj == LNULL` markiert Header-Sentinels; niemals dereferenzieren.
4. `m_GColList`/`m_GRowList` sind nach `m_GColPos`/`m_GRowPos` aufsteigend sortiert.
5. Nach InsertColBefore/Remove werden ALLE folgenden Spalten umnummeriert UND ihre Pixel-StartPos neu berechnet.
6. GOIns liefert FALSE wenn (x, y) belegt; macht den Spalten-Eintrag rollbacken bevor es FALSE liefert.
7. RemoveCol/Row prüft IsColEmpty/IsRowEmpty; no-op wenn nicht leer.
8. Die statischen `s_setPosFree`/`s_setBlocks` sind global über alle Grid-Instanzen geteilt — in TS irrelevant, aber dokumentieren.
