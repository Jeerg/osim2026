<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Internet → GCP-Projekt | Externe Auth-Requests, GCS-Signed-URLs |
| Cloud Run → Cloud SQL | Service-Account-Auth, Private IP |
| Cloud Run → Secret Manager | Service-Account mit roles/secretmanager.secretAccessor |
| Terraform-Operator → GCP | Admin-Credentials des Users, sollten nicht im Repo |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-04-01 | I (Information Disclosure) | terraform state | mitigate | State in GCS-Bucket mit uniform_bucket_level_access, IAM nur für Owner |
| T-04-02 | E (Elevation) | Service-Accounts | mitigate | Minimal-Rollen (roles/cloudsql.client statt cloudsql.admin), keine roles/owner |
| T-04-03 | I | DB-Passwort in tfstate | mitigate | random_password + sensitive=true im Output, Secret Manager als Lese-Pfad für Anwendungen |
| T-04-04 | D (DoS) | Cloud SQL Public IP | mitigate | Nur Private IP, kein public endpoint, Cloud SQL Proxy für lokales Dev |
| T-04-05 | T (Tampering) | tfstate | mitigate | Versioning auf State-Bucket aktivieren (kein Modul-Item, im Bootstrap-Schritt 4) |
| T-04-06 | R (Repudiation) | IAM-Änderungen | accept | Audit-Logging ist GCP-default aktiv; bei Compliance-Anforderungen später Org-Policy |
</threat_model>
