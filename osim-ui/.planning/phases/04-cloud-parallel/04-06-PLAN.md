Migration-Strategie:
- Cloud SQL Proxy als sidecar container im Build-Step:
  step1: pull cloud-sql-proxy, start in background, port 5432.
  step2: alembic upgrade head mit DATABASE_URL=postgresql+asyncpg://...@127.0.0.1:5432/osim.
- Per-Tenant-Schema-Migrationen: Alembic-env.py iteriert über alle tenant_*-Schemas
  und führt op.execute für jeden aus. Phase 1 hat hier hoffentlich schon Convention
  gesetzt — wenn nicht, in scripts/migrate.sh nach upgrade head zusätzlich Loop.

Firebase Hosting:
- Portal-Build: cd portal && npm ci && npm run build → portal/dist/.
- Deploy via firebase-tools: firebase deploy --only hosting:portal --project=$_PROJECT_ID
  --non-interactive.
- portal/firebase.json: rewrites / → /index.html (SPA), public=dist, ignore standard.
- portal/.firebaserc: { "projects": { "default": "osim-ui-staging", "prod": "osim-ui-prod" } }.

Trigger:
- google_cloudbuild_trigger für jeden der drei Builds, pro Env (staging vom main-
  branch, prod vom tag v*).
- 6 Trigger insgesamt:
  * api-staging (branch=^main$)

  * portal-staging (branch=^main$, included_files=portal/**)

    scripts/migrate.sh:
    ```
    #!/usr/bin/env bash
    set -euo pipefail
    INSTANCE="${1:?usage: migrate.sh <instance_connection_name>}"
    DB_USER="${DB_USER:-osim-app}"
    DB_PASS="${DB_PASS:?env DB_PASS required}"
    DB_NAME="${DB_NAME:-osim}"
    
    cloud-sql-proxy "$INSTANCE" --port 5433 &
    PROXY_PID=$!
    trap "kill $PROXY_PID" EXIT
    sleep 5
    
    export DATABASE_URL="postgresql+asyncpg://${DB_USER}:${DB_PASS}@127.0.0.1:5433/${DB_NAME}"
    alembic -c alembic.ini upgrade head
    ```
    
    cloudbuild-api.yaml:
    ```
    substitutions:
      _ENV: staging
      _REGION: europe-west3
      _CLOUDSQL_INSTANCE: ${PROJECT_ID}:${_REGION}:osim-pg-${_ENV}
    
    steps:
      # 1. Lint + Test
      - name: gcr.io/cloud-builders/docker
        id: test
        entrypoint: bash
        args: ['-c', 'docker build --target builder -t test . && docker run --rm test uv run pytest -x']
    
      # 2. Build runtime image
      - name: gcr.io/cloud-builders/docker
        id: build
        args: ['build', '-t', 'gcr.io/$PROJECT_ID/osim-api:$SHORT_SHA',
               '-t', 'gcr.io/$PROJECT_ID/osim-api:latest', '.']
    
      # 3. Push
      - name: gcr.io/cloud-builders/docker
        id: push
        args: ['push', '--all-tags', 'gcr.io/$PROJECT_ID/osim-api']
    
      # 4. Migrate (mit Secret aus Secret Manager als env)
      - name: gcr.io/google-appengine/exec-wrapper
        id: migrate
        entrypoint: bash
        env: ['DB_USER=osim-app', 'DB_NAME=osim']
        secretEnv: ['DB_PASS']
        args: ['-c', 'apt-get update && apt-get install -y curl &&
                     curl -o /usr/local/bin/cloud-sql-proxy https://storage.googleapis.com/cloud-sql-connectors/cloud-sql-proxy/v2.11.0/cloud-sql-proxy.linux.amd64 &&
                     chmod +x /usr/local/bin/cloud-sql-proxy &&
                     bash scripts/migrate.sh ${_CLOUDSQL_INSTANCE}']
        # Besser: dediziertes Image (siehe cloudbuild-migrate.yaml unten).
    
      # 5. Deploy auf Cloud Run
      - name: gcr.io/cloud-builders/gcloud
        id: deploy
        args: ['run', 'deploy', 'osim-api-${_ENV}',
               '--image=gcr.io/$PROJECT_ID/osim-api:$SHORT_SHA',
               '--region=${_REGION}',
               '--service-account=osim-api-sa@$PROJECT_ID.iam.gserviceaccount.com',
               '--no-allow-unauthenticated',
               '--set-env-vars=ENV=${_ENV},EVENT_BACKEND=pubsub,ORCHESTRATOR_BACKEND=cloudtasks',
               '--set-secrets=DATABASE_URL=osim-database-url:latest,FIREBASE_ADMIN_KEY=osim-firebase-admin-key:latest',
               '--add-cloudsql-instances=${_CLOUDSQL_INSTANCE}',
               '--min-instances=1', '--max-instances=10',
               '--cpu=2', '--memory=2Gi']
    
    availableSecrets:
      secretManager:
        - versionName: projects/$PROJECT_NUMBER/secrets/osim-database-url/versions/latest
          env: DB_PASS  # nur das Passwort, nicht die ganze URL; URL wird aus DB_USER+DB_PASS+CLOUDSQL_INSTANCE gebaut
    
    options:
      logging: CLOUD_LOGGING_ONLY
      machineType: E2_HIGHCPU_8
    
    timeout: 1800s
    ```
