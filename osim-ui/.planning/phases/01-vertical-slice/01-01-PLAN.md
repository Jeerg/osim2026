<objective>
Welle 0 von Phase 1: Engine-Vorarbeit.

Die osim-engine hat heute nur einen OTX-Reader/Loader, KEINEN Writer. Ohne Writer ist der in CONTEXT.md D-01/D-02 festgelegte Save-back-Pfad (Browser-Edit → JSON-Tree → Server → PSimulator → OTX-Text → Storage) nicht möglich. Dieser Plan implementiert den Writer im osim-engine-Repo (NICHT in osim-ui), spiegelt die Klassen-Registry des Loaders und stellt Roundtrip-Stabilität mit den vorhandenen Beispielmodellen sicher.

Purpose: Phase 1 ist als Modellierungs-Werkzeug konzipiert (D-01..D-04, D-10..D-14). Ohne dump_simulator_to_otx kein Save-back, kein Versionierung, keine sinnvolle Edit-Story. Welle 0 muss komplett grün sein, bevor irgendein UI-Code etwas zu speichern hat.

Output:
- `osim_engine/io/otx_writer.py` mit `dump_simulator_to_otx(sim) -> str`
- Roundtrip-Tests gegen Dummy.otx und Fertigungsstruktur1_mit_AslFj.otx
- Re-Export über `osim_engine.io`

Format-Struktur (Latin-1, Header-Zeile + N Objekt-Zeilen):
```
OIDArray|N!
#ClassName|attr;val|attr;val|...|$N;sub1;sub2;...|!
```

Wesentliche OtxObject-Felder (aus otx_reader.py):
```python
@dataclass
class OtxObject:
    oid: int
    klass: str
    props: dict[str, str]     # einfache Attribute attr -> val
    sub_refs: list[int]       # $N;... Referenzen
    # (genaue Felder bitte aus otx_reader.py verifizieren)
```

LoadResult (aus otx_loader.py):
```python
@dataclass
class LoadResult:
    simulator: PSimulator
    otx: OtxFile
    instances: dict[int, Any]     # oid -> py_obj
    loaded: Counter[str]          # klass -> count
    skipped: Counter[str]
    unsupported: Counter[str]
    warnings: list[str]
    coverage_ratio: float
```

<risks>
- **Format-Reverse-Engineering-Fehler:** otx_reader.py muss vor Implementierung in voller Tiefe gelesen werden. Trennzeichen-/Escape-Schema-Mismatch macht den Roundtrip wertlos. Mitigation: erste Test-Iteration mit `assert reader_output_after_dump == reader_output_before_dump` für ein Mini-Modell (1 Auslöser + 1 Knoten), erst dann auf Dummy.otx skalieren.
- **OID-Stabilität:** Wenn der Reader OIDs aus dem File übernimmt, der Writer aber neu vergibt, kann der Roundtrip semantisch korrekt aber strukturell anders aussehen. Test-Strategie: Identität auf Klass/Props-Ebene prüfen, nicht auf OID-Ebene.
- **Fixtures-Verfügbarkeit:** Dummy.otx und Fertigungsstruktur1_mit_AslFj.otx liegen im OSim2004-Repo, nicht im osim-engine-Repo. conftest.py muss sauber skippen, wenn nicht erreichbar — sonst bricht CI.
- **Unsupported-Passthrough korrumpiert File:** Wenn ein Skip-Objekt im Original beschädigte sub_refs hat, propagiert der Writer die Beschädigung. Mitigation: `include_unsupported_passthrough=True` default-an für Phase-1-Save-back; für JSON-Tree-only-Edit später opt-out via Flag.
</risks>
