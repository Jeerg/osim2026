### Task 2 — Neuer PRessMengeMatrixViewer + Loeschung Legacy + Registry-Update

Neue Datei `osim-ui/portal/src/viewers/PRessMenge/PRessMengeMatrixViewer.tsx` (+645 LOC).

#### 2D-Matrix-Aufbau

- **Zeilen:** Alle `PRessMenge`-Objekte aus `allObjects`, sortiert nach OID.
- **Spalten:** `PDlplKnoten` des aktuellen Durchlaufplans, gelesen aus `obj.attrs.m_lKnoten` (V1-Pragma: inline-Array) oder `obj.sub_refs[0]` (Plan-1.2-Wire). Keine Klass-Filterung — 1:1 C++ Z.452-470, wo `m_lKnotenOber != ONULL` der einzige Filter ist.
- **Cells:** PAssozMenge*-Instanz mit Mengen-Wert. cellMap-Build iteriert ueber `col.m_lAssozRess.sub_refs[0]`, filtert auf `PASSOZMENGE_KLASSEN`-Set, liest `m_lMengRess` als Scalar-Pointer (NICHT Wrapper — AUDIT 4.4) und das subklassen-spezifische Mengen-Attr via `extractMenge(assoz)`.

#### Interaktion (1:1 zum C++-Original)

| User-Aktion | C++ Z. | Effekt |
|---|---|---|
| Linksklick auf leere Cell | `OnLButtonDown` Z.867-882, `GetAssozMenge` Z.787-833 | Lazy-Create `PAssozRessourceLList` am Knoten (falls noch nicht), dann neue `PAssozMenge<Typ>` mit `m_sName` + `m_lMengRess` + `typeDef.mengen_attr` = Toolbar-Menge, `appendSubRef(wrapper, 0, neueAssoz)` |
| Linksklick auf belegte Cell | C++ Z.867 `if (gobj == NULL)` | No-Op (faellt aus dem Branch) |
| Rechtsklick auf belegte Cell | `OnRButtonDown` Z.985 `assmenge.Delete()` | `removeSubRef(wrapper, 0, assoc.oid)` + `deleteObject(assoc.oid)` |
| Ctrl/Shift+Klick | C++ Z.842 `if (nFlags&MK_CONTROL) return;` | Block-Selektion, kein Paint |
| Ctrl+C / Ctrl+V | Document-Level Listener | Block-Copy/Paste mit `ClipboardCellPayload = { klass, menge }`. Bestehende Cells werden NICHT ueberschrieben |

#### Status-Pill pro Subklasse

```
PAssozMengeErzgt          → status-finished-bg (gruen, "E <menge>")
PAssozMengeVerbr          → status-failed-bg   (rot,   "V <menge>")
PAssozMengeVerbrZwischen  → status-aborted-bg  (amber, "Z <menge>")
PAssozMengeAbfr           → color-primary-light (blau, "A <menge>")
```
