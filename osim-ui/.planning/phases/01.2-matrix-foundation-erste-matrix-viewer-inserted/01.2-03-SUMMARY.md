# Phase 01.2 Plan 03: Matrix-Foundation Welle B Summary

Foundation-Welle 1.2-B der Matrix-Foundation: `useInlineCellEdit`-Hook für
das Excel-Style-Edit-Pattern (Doppelklick → Input → Enter committet, ESC
verwirft, Blur committet — bei Enum/Bool: onChange committet, ESC verwirft).
Lift der bisher in `portal/src/viewers/PRess/matrix-common.tsx::renderEditCell`
(Z.126-205) verbauten Logik in einen wiederverwendbaren Hook, der von
allen Matrix-Viewer-Konsumenten (PRessBelegMatrixViewer in Welle 1.2-E,
PRessMengeMatrixViewer etc.) übernommen werden kann. Drei Editor-Typen
bleiben erhalten: Variable (shadcn Input), Enum (native select), Bool
(checkbox). Der Parser ist als Pure-Function `parseRaw` exportiert und
direkt unit-testbar ohne Hook-Mount — int/float-NaN landet konservativ
auf 0 (Schutz vor Engine-Crash bei numerischen OSim-Feldern). Der Hook
ist foundation-agnostisch (CONTEXT D-4): kennt weder Wire-Schemata noch
PropertySchema noch Lock-Status — Konsument durchreichen `octrlType`,
`valueType`, `enumValues`, `readOnly` und die Commit-/Cancel-Callbacks.
14 Vitest-Specs laufen grün, tsc bringt 0 neue Errors für die Foundation-
Matrix-Files, ESLint ist 0 Errors, Full-Suite 34 Files / 212 passed (+14
neu, keine Regression).

### Task 1: useInlineCellEdit + parseRaw + 14 Specs (RED Commit `6ab458e`, GREEN Commit `2212e90`)

**Was gebaut:**

- `portal/src/graph/foundation/matrix/useInlineCellEdit.tsx` (274 LOC):
  - **Pure-Function** `parseRaw(raw, octrlType, valueType?)` — Lift aus
    `matrix-common.tsx` Z.174-186. Regeln:
    - `Enum` → `Number(raw)`
    - `Bool` → `Boolean(raw)`
    - `Variable+int`: `""`/null → 0, sonst `parseInt(raw, 10)`, NaN → 0
    - `Variable+float`: `""`/null → 0, sonst `parseFloat(raw)`, NaN → 0
    - `Variable+string` (default): raw unverändert
  - **Hook** `useInlineCellEdit<TVal>(opts)` mit Result `{ editing,
    startEdit, commit, cancel, EditorElement }`.
    - `startEdit` no-op bei `readOnly=true`.
    - `cancel` setzt `editing=false` + ruft `onCancel?.()`.
    - `commit(raw)` setzt `editing=false` + ruft
      `onCommit(parseRaw(raw, octrlType, valueType))`.
    - `EditorElement` ist `null` solange `!editing`, sonst eines der drei
      Editor-Elemente.
  - **Helper** `buildEditor(opts, commit, cancel)`:
    - `Enum + enumValues` → `<select autoFocus defaultValue={…}
      onChange={commit} onBlur={cancel} onKeyDown(ESC)={cancel}
      className="h-9 w-full rounded-md border border-input bg-card px-2 text-sm">{enumValues.map(option)}</select>`
    - `Bool` → `<input type="checkbox" autoFocus
      defaultChecked={Boolean(value)} onChange={(e)=>commit(e.target.checked)}
      onBlur={cancel} onKeyDown(ESC)={cancel} className="h-4 w-4" />`
    - default (Variable): `inputType = "text" | "number"`,
      `step = "any" | "1" | undefined`. Render `<Input autoFocus
      type={inputType} step={step} defaultValue={…}
      onBlur={(e)=>commit(e.currentTarget.value)}
      onKeyDown={Enter→commit, ESC→cancel}
      className="h-8 bg-card focus:ring-2 focus:ring-primary" />`
  - Cleanup: keine globalen Listener — alle Event-Handler hängen am
    gerenderten Element, Render-Tree-Unmount räumt automatisch auf.

### Auto-fixed Issues

**1. [Rule 3 - Blocker] Datei-Endung .ts → .tsx**

- **Found during:** GREEN-Phase, vor erstem Test-Lauf
- **Issue:** PLAN gibt die Datei als `useInlineCellEdit.ts` vor (siehe
  PLAN-Frontmatter `files_modified`, `<read_first>`, `<verify>`-Block).
  Der Hook enthält aber JSX (drei Editor-Elemente: `<Input>`, `<select>`,
  `<input type="checkbox">`). TypeScripts strict mode akzeptiert JSX-
  Syntax ausschließlich in `.tsx`-Dateien — die Verify-Pflicht
  `npx tsc --noEmit src/graph/foundation/matrix/useInlineCellEdit.ts`
  wäre mit `.ts`-Endung garantiert gescheitert.
- **Fix:** Datei direkt nach Erstellung umbenannt:
  `mv useInlineCellEdit.ts useInlineCellEdit.tsx`. Im Barrel-Export
  reicht der modul-resolution-Name `useInlineCellEdit` (ohne Endung) —
  konsumentenseitig keine Änderung. Die Spec hatte ohnehin schon
  `useInlineCellEdit.spec.tsx` (passend zur Endung).
- **Files modified:** `portal/src/graph/foundation/matrix/useInlineCellEdit.tsx`
  (statt .ts), `index.ts` (Barrel — auf den Modul-Namen importiert ohne
  Endung, keine inhaltliche Konsequenz)
- **Begründung:** Die anderen JSX-tragenden Foundation-Files in derselben
  Direktive (MatrixGrid.tsx, MatrixCell.tsx) sind ebenfalls `.tsx` — der
  PLAN war hier inkonsistent (vermutlicher Tippfehler).
- **Commit:** Beide Korrekturen sind innerhalb des GREEN-Commits `2212e90`.

**2. [Rule 1 - Bug] Spec Test 13 — DOM-Lifecycle-Mismatch**

- **Found during:** Erster GREEN-Test-Lauf (13 von 14 grün)
- **Issue:** Die ursprüngliche Spec für Test 13 testete im selben Render-
  Block zuerst `onChange→commit` und dann `keyDown(Escape)→cancel`. Nach
  dem commit ist aber `editing=false`, `EditorElement=null` und das
  `<select>` ist nicht mehr im DOM. Der subsequente `fireEvent.keyDown`
  trifft das (bereits unmounted) Element ohne neuen ESC-Handler →
  `onCancel` wird nie gerufen → Test failt.
- **Fix:** Test 13 in zwei isolierte Render-Blöcke gesplittet — 13a
  (onChange→commit) mit eigenem mount/unmount, 13b (ESC→cancel) mit
  eigenem mount/unmount. Analog Test 12 (Variable), der bereits
  dreigeteilt war.
- **Files modified:** `portal/src/graph/foundation/matrix/__tests__/useInlineCellEdit.spec.tsx`
- **Commit:** GREEN-Commit `2212e90` (vor Push).
- **Begründung:** Spec-Hygiene "ein Render-Cycle = eine logische
  Interaktion" — derselbe Pattern wie der Plan ihn implizit für Test 12
  vorsieht (12a/12b/12c).

**3. [Rule 2 - Auto-Add] useRef-Pattern → direkte useCallback-Memoization**

- **Found during:** ESLint-Lauf nach erstem GREEN-Grün
- **Issue:** Die initiale Hook-Implementation nutzte ein `optsRef`-
  Indirektionspattern (analog `useAutoSave.ts::mutateRef`), um
  `commit`/`cancel`-Identität stabil zu halten. ESLint
  `react-hooks/refs` hat zwei Errors gemeldet:
  `Cannot access refs during render` — weil `useMemo`-Callbacks im
  Render-Body laufen und das `optsRef.current` dort gelesen wurde
  (über die mitgegebenen `commit`/`cancel`-Closures).
- **Fix:** `optsRef` entfernt, direkte `useCallback`-Memoization auf
  destructured opts-Felder (`readOnly`, `onCancel`, `onCommit`,
  `octrlType`, `valueType`). Identisches Verhalten, marginal mehr
  Closure-Allocations, dafür lint-clean. Der Hook hält ohnehin kein
  langlebiges Resource (kein Interval, kein WS, kein Subscriber), das
  die ref-Indirektion gerechtfertigt hätte.
- **Files modified:** `portal/src/graph/foundation/matrix/useInlineCellEdit.tsx`
- **Commit:** GREEN-Commit `2212e90`.
