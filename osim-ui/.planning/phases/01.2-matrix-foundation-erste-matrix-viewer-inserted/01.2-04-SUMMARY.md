für Block-Select-Mechanik nach CONTEXT D-5 (Anchor + Focus, Range,

### Auto-fixed Issues

**1. [Rule 1 - Bug] Test 16 fireEvent-Target-Override**

- **Found during:** Erster GREEN-Test-Lauf (17 von 18 grün)
- **Issue:** Ursprünglich verwendet der Spec für Test 16
  `fireEvent.pointerMove(window, { target: targetCell })`. Testing-
  Library setzt aber `e.target = window` bei einem `fireEvent` auf
  window — der Init-Override via `{target: ...}` greift nicht
  vollständig (in der `dispatchEvent`-Phase wird target je nach
  Event-Init aus dem dispatcher-Target gezogen). Folge: der globale
  `pointermove`-Listener im Hook sieht `target=window`, der `closest`-
  Aufruf liefert null, und `setSelection` wird nicht aufgerufen —
  Test schlägt mit `{startRow:2, endRow:2, ...}` statt `{2, 5, 2, 7}`
  fehl.
- **Fix:** `fireEvent` direkt am Cell-Element (`fireEvent.pointerMove(targetCell)`);
  da `pointermove` per Default bubblet, fängt der Window-Listener das
  Event über die Bubble-Phase ein UND `e.target` ist dann korrekt das
  Cell-Element. Inline-Kommentar im Test (4 Zeilen) erklärt den Grund.
- **Files modified:** `portal/src/graph/foundation/matrix/__tests__/useBlockSelection.spec.tsx` (Test 16, 4 Zeilen Doku + 1 Zeile Fix)
- **Commit:** Im GREEN-Commit `33d3935` enthalten — kein separater
  Commit nötig, weil Production-Code unverändert blieb und der Fix nur
  die Test-Verkabelung korrigiert.

**2. [Rule 3 - Blocker] npm install im Worktree (487 packages)**
